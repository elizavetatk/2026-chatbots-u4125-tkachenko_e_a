# Telegram-бот: гид по Русскому музею

Простой Telegram-бот на Python (`python-telegram-bot`) для мини-экскурсий по картинам Русского музея (Санкт-Петербург).

## Возможности

- Команда `/start` приветствует пользователя и предлагает начать экскурсию кнопкой `Да!`
- Пошаговый диалог через `ConversationHandler`:
  - выбор формата экскурсии (`Короткая` / `Подробная`)
  - выбор интереса (`История`, `Сюжет`, `Биография`, `Живопись`)
- Генерация экскурсии из 3 случайных картин
- Для каждой картины показывается:
  - название
  - художник
  - год
  - текст-заглушка по выбранному интересу
- Оценка экскурсии от 1 до 5
- Предложение начать новую экскурсию после оценки
- Обработка ошибок (глобальный `error_handler`)

## Структура проекта

- `bot.py` — код бота
- `paintings.json` — данные о 10 картинах
- `requirements.txt` — зависимости
- `.env.example` — пример переменной окружения

## Требования

- Python 3.10+

## Установка и запуск

1. Создайте и активируйте виртуальное окружение:

```bash
python -m venv .venv
```

Windows (PowerShell):

```powershell
.venv\Scripts\Activate.ps1
```

2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` на основе примера:

```bash
copy .env.example .env
```

4. Впишите токен бота в `.env`:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
```

5. Запустите бота:

```bash
python bot.py
```

## Как пользоваться

1. Откройте чат с ботом в Telegram.
2. Отправьте `/start`.
3. Нажмите `Да!`.
4. Выберите тип экскурсии и интерес.
5. Прочитайте 3 карточки с картинами.
6. Отправьте оценку (`1`–`5`).
7. При желании начните новую экскурсию.

## Примечания

- Данные хранятся в `paintings.json` (временное решение, без базы данных).
- Сейчас описания картин — шаблонные заглушки.
- Бот не использует `python-dotenv`: токен берется из системной переменной `TELEGRAM_BOT_TOKEN` или напрямую из файла `.env`.
