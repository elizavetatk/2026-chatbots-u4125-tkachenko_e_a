# Telegram-бот-гид (lab_2)

Обновленная версия бота-гида по Русскому музею с загрузкой описаний картин из CSV.

## Что умеет бот

- Команда `/start` приветствует пользователя и предлагает начать экскурсию кнопкой `Да!`
- Пошаговый опрос на кнопках:
  - длина экскурсии: `Короткая` или `Подробная`
  - интерес: `История`, `Сюжет`, `Биография`, `Живопись`
- Случайно выбирает 3 картины из CSV
- Для каждой картины выводит:
  - название
  - художника
  - описание нужного типа (по длине + интересу)
- Если нужное поле пустое, используется fallback: первое доступное описание этой картины
- После экскурсии просит оценку от `1` до `5` и предлагает начать новую

## Формат CSV

Файл `paintings.csv` должен содержать колонки:

- `title`
- `artist`
- `short_history`, `short_plot`, `short_biography`, `short_painting`
- `detailed_history`, `detailed_plot`, `detailed_biography`, `detailed_painting`

Файл с примером на 10 реальных картин Русского музея уже добавлен в проект.

## Структура папки

- `bot2.py` — код бота
- `paintings.csv` — данные и тексты описаний
- `.env.example` — пример переменной окружения
- `requirements.txt` — зависимости

## Установка и запуск (Windows, PowerShell)

1. Перейдите в папку проекта:

```powershell
cd "C:\Users\Liza\Documents\ITMO\Vibe_Coding\lab_2_bot"
```

2. Создайте и активируйте виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

3. Установите зависимости:

```powershell
pip install -r requirements.txt
```

4. Создайте `.env` на основе примера:

```powershell
Copy-Item .env.example .env
```

5. Укажите токен в `.env`:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
```

6. Запустите бота:

```powershell
python bot2.py
```

## Обработка ошибок

- Проверка наличия токена (`TELEGRAM_BOT_TOKEN`)
- Проверка наличия CSV и обязательных колонок
- Проверка, что в CSV есть валидные строки
- Глобальный обработчик ошибок Telegram (`error_handler`)
