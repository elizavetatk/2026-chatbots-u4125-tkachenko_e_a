У меня есть работающий Telegram-бот (файл bot.py в папке lab_1_bot).
Улучши моего Telegram-бота-гида, добавив работу с CSV-файлом.

Текущий функционал:
- Предлагает экскурсию
- Бот спрашивает длину экскурсии (короткая / подробная) и интерес (история / сюжет / биография / живопись)
- Выдает 3 случайные картины, но пока с текстами-заглушками
- Просит оценить экскурсию цифрой от 1 до 5
- Предлагает новую экскурсию

Новый функционал:
- Бот должен загружать описания картин из CSV-файла
- В CSV должны быть колонки:
  - title (название картины)
  - artist (художник)
  - short_history, short_plot, short_biography, short_painting (короткие описания по 3-4 предложения)
  - detailed_history, detailed_plot, detailed_biography, detailed_painting (подробные описания по 5-7 предложений)
- При формировании экскурсии бот:
  - Случайно выбирает 3 картины
  - В зависимости от выбора пользователя (длина экскурсии → short/detailed, интерес → history/plot/biography/painting) берет нужный текст
  - Выдает пользователю: название + художник + соответствующий текст
- Если для выбранной картины нет описания по нужному интересу — бот использует описание по умолчанию (например, общее описание картины)
- Данные загружаются при старте бота
